## Overview

This is a **Traefik middleware plugin** written in Go. It extracts real client IPs from requests proxied through Cloudflare (proxy or tunnel) by trusting Cloudflare server IPs and rewriting headers accordingly.

Module path: `github.com/enoqv/cloudflarewarp`
Go version: 1.18

## Commands

```bash
# Run unit tests
go test -race -coverprofile=coverage.txt -covermode=atomic ./*_test.go

yaegi test .

# Update bundled Cloudflare IP ranges from live Cloudflare API

```

## Architecture

The plugin is a single Go package with two source files:

- **`cloudflarewarp.go`** — Main plugin logic. Implements the Traefik plugin interface (`New`, `ServeHTTP`). On each request, it checks if `req.RemoteAddr` falls within a trusted CIDR range. If trusted (i.e., coming from Cloudflare), it rewrites headers using Cloudflare's own headers. If not trusted, it strips CF headers and sets `X-Real-IP` to the direct connection IP.

- **`ips/ips.go`** — Hardcoded list of Cloudflare IP ranges (both IPv4 and IPv6). This file is auto-generated by `updateCFIps.sh` which fetches current IPs from Cloudflare's public API. Run the script and commit the result when IPs need updating.

### Header rewriting logic

| Condition | `X-Is-Trusted` | `X-Real-IP` | `X-Forwarded-For` | `X-Forwarded-Proto` |
|---|---|---|---|---|
| Trusted (from CF) | `yes` | CF-Connecting-IP | CF-Connecting-IP | from CF-Visitor JSON |
| Trusted + bad CF-Visitor JSON | `danger` | unchanged | unchanged | unchanged |
| Not trusted | `no` | direct IP | unchanged | unchanged |
| Parse error | HTTP 400/500 | — | — | — |

### Plugin configuration

The `Config` struct has two fields:
- `trustip` — additional CIDR ranges to trust beyond the built-in CF list
- `disableDefault` — set `true` to skip the bundled Cloudflare IPs in `ips/ips.go`

The `.traefik.yml` file defines plugin metadata for the Traefik plugin registry.

### Testing structure

- `cloudflarewarp_test.go` — unit tests using `httptest`, covering trusted/untrusted/IPv6/malformed cases
- `test/` — Docker-based integration tests using `traefik/whoami`, run with both TOML and YAML config formats via `test.sh`
